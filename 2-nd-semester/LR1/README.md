# Отчет по выполнению лабораторной работы №1: Развертывание статического сайта на Hugo с использованием SourceCraft

## 1. Цель работы
Реализовать CI/CD-процесс публикации статического сайта на базе генератора Hugo с использованием DevOps-платформы SourceCraft. В рамках работы необходимо настроить хостинг, автоматизировать сборку и развертывание, а также дополнить пайплайн проверками синтаксиса Markdown.

## 2. Выполненные шаги

### 2.1. Регистрация и создание организации
1. **Регистрация:** Выполнена аутентификация в сервисе SourceCraft с использованием учетной записи Yandex.
2. **Создание организации:** Создана публичная организация. Это обязательное условие, так как хостинг статического сайта возможен только из публичного репозитория, принадлежащего публичной организации.

### 2.2. Создание репозитория
1. В созданной публичной организации был создан **публичный репозиторий**. Название репозитория: `lab1-hugo`.
2. Репозиторий создан на основе шаблона `sites-landing` (как рекомендовано в [инструкции](https://sourcecraft.dev/portal/docs/ru/sourcecraft/tutorials/sites)), который содержит предустановленную конфигурацию SourceCraft Sites и типовые файлы проекта.

### 2.3. Настройка доступа
Для работы с репозиторием были созданы необходимые токены и ключи:
1. **Персональный токен (PAT)** для работы по HTTPS. Токен создан согласно [документации по PAT](https://sourcecraft.dev/portal/docs/en/sourcecraft/security/pat). (Доступ к этой странице документации был ограничен, но процесс описан в [общей инструкции по хостингу](https://sourcecraft.dev/portal/docs/ru/sourcecraft/tutorials/sites)).
2. **SSH-ключи** для безопасного доступа по протоколу SSH. Генерация и добавление ключа выполнены по [руководству](https://sourcecraft.dev/portal/docs/en/sourcecraft/security/ssh).

### 2.4. Клонирование и первоначальный push
1. Репозиторий был склонирован на локальную машину с использованием созданного PAT или SSH-ключа.
2. В репозиторий была добавлена базовая структура для статического сайта на Hugo. В качестве основы использован шаблон, предоставленный платформой, с последующей адаптацией под Hugo.
3. Изменения были закоммичены и запущены в основной ветке (`main`) репозитория SourceCraft.

### 2.5. Реализация CI/CD-процесса

#### 2.5.1. Обзор существующего пайплайна
Платформа SourceCraft использует файл конфигурации `.sourcecraft/ci.yaml` для описания CI/CD-процессов. По умолчанию в шаблоне `sites-landing` был определен рабочий процесс `build-site`, который:
- Собирает статический сайт из исходных файлов в директории `src`.
- Публикует сгенерированные файлы в директорию `site` в ветке `release`.
- Автоматически запускается при внесении изменений в основную ветку `main`.


#### 2.5.2. Итоговый пайплайн
Полный файл .sourcecraft/ci.yaml включает следующие этапы:

Checkout: Клонирование кода.

Setup Hugo: Установка Hugo.

Lint Markdown: Проверка синтаксиса Markdown.

Check Links: Проверка ссылок в Markdown.

Build: Сборка сайта с помощью Hugo.

Deploy: Публикация собранного сайта в ветку release.
